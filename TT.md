# **Цель работы**

Получение практических навыков в проектировании, разработке и обучении нейросети с использованием языка разработки Python с использованием Keras или TensorFlow.  

## <u>Раздел 1:</u>

- Выбрать задачу для разработки нейросети. Тематика задачи - по выбору студента (в разумных пределах).

Пример 1: разработка сверточной нейросети, предназначенной для поиска участков опустынивания сельхозземель по данным съемки ДЗЗ. (В подобной задаче вместо снимков ДЗЗ, которых у вас точно не будет, можно самостоятельно собрать наборы изображений из ГИС-систем типа GoogleMaps или Яндекс-карты)

Пример 2: разработка нейросети, предназначенной, например, для оценки состояния здоровья (например, космонавта перед полетом) по данным анализов крови. Открытые (обезличенные) наборы данных с результатами анализов крови можно найти в сети интернет, на сайтах типа  Kaggle или GitHub и т.д., а можно найти справочник по анализам крови и сгенерировать обучающий набор самостоятельно.

Пример 3: разработка сверточной нейросети, предназначенной для классификации типов автомобилей по фотографии. Набор изображний авто разных типов предполагается сформировать самостоятельно по данных из сети интернет

и т.д.

- Необходимо подробно описать задачу для нейросети (фактически тут вы должны сформировать полноценное техническое здание на разработку нейросети на языке Python, которую будете разработывать в лабораторном проекте)

- Также в этом разделе необходимо провести обоснование выбора типа нейросети в вашем проекте (например, сверточной нейросети CNN или рекуррентной неросети LSTM) Также тут необходимо описать выбранные параметры нейросети, как например, количество слоев нейронов в нейросети, количество неронов в слое, функции активации и т.д.

## <u>Раздел 2:</u>
- Подготовка массива исходных данных, предназначенных для обучения нейросети. Исходные данные можно найти в сети интернет или сформировать самим. 

- В процессе подготовки массива исходных данных рекомендуется использовать дополнительные библиотеки Pandas или Pillow языка Pyhon.

- Необходимо представить подробное описание исходных данных (и указать их источник в сети интернет).

- ВНИМАНИЕ!!! Требование к исходным данным: Исходные данные для обучения НС могут браться только из открытых источников в сети интернет, или сформированы самостоятельно на основе данных из открытых источников. При этом недопускается использовать реальные данные с вашей работы !!!. (даннное условие было утверждено в 2025 году на заседании кафедры 604, проводящей данный курс лекций по Т.И.И.  И в случае нарушения данного условия студентом работа вообще не будет приниматься к рассмотрению)

## <u>Раздел 3:</u>

- Разработка программного кода нейросети.

- Требование - используется язык разработки Python.

- Для разработки кода нейросети могут использоваться специализированные библиотеки TensorFlow, Keras, PyTorch языка Python.

## <u>Раздел 4:</u>

- Обосновать выбранный тип и параметры обучения вашей нейросети.

- Провести обучение разработанной вами в разделе 3 нейросети на основе подготовленного в разделе 2 набора исходных данных.

- Провести анализ результатов обучения вашей нейросети / продемонстрировать результат работы обученной нейросети.

## <u>Раздел 5:</u>

- Разработать интерфейс для удобства работы пользователя с нейросетью. Для решения указанной задачи рекомендуется использовать любую из указанных библиотек языка Python DASH, FLASK, Django (в данном случае предполагается локальный запуск приложения в браузере на вашем ноутбуке) или TKinter, PyQt.

## <u>Раздел 6:</u>

- Написать подробный отчет по всем этапам работы.

- В отчете необходимо оисать задачу (ТЗ), обоснование выбора структуры нейросети, подготовку исходных данных для обучения, программный код на языке Python (полностью), процес обучения нейросети, полученный результат, выводы по работе.

- Учитывая объем поставленной задачи, итоговый отчет должен иметь объем не менее 50 страниц + Титульный лист. 

Также с учетом объема поставлееной задачи допускается выполнение лабораторного проекта подгруппами из 2х или 3х студентов. В этом случае итоговый отчет необходимо дополнить разделом, подробно(!) описывающим вклад каждого из студентов подгруппы в общий проект.
